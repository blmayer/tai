tai.nvim


a minimal, dependency-free Neovim plugin that brings
AI coding agents directly to your editor.





features

- dual-agent architecture: planner coordinates,
  coder implements. each has independent history.
- session persistence: conversations auto-save and
  restore between Neovim sessions.
- task tracking: agents use todos and notes tools
  to stay organized on long tasks.
- file read/write, shell commands, image analysis.
- rate limiting (rpm/tpm) to stay within API quotas.
- streaming and non-streaming modes.
- code folding in the chat buffer.
- provider-side tool pass-through (web_search etc).
- configurable shell command safety allowlist.


providers

- gemini         GEMINI_API_KEY
- groq           GROQ_API_KEY
- minimax        MINIMAX_API_KEY
- mistral        MISTRAL_API_KEY
- ollama         (local, localhost:11434)
- llama.cpp      (local, localhost:8080)
- openai         OPENAI_API_KEY
- openai resp.   OPENAI_API_KEY (Responses API)
- openrouter     OPENROUTER_API_KEY
- stepfun        STEPFUN_API_KEY
- xai            XAI_API_KEY
- z.ai           Z_AI_API_KEY
- custom         (set options.url)


agent tools

- read       read files (with line ranges)
- shell      run commands in the project root
- edit       search-and-replace (with multi flag)
- write      create new files
- send_image send images for visual analysis
- coder      planner delegates to coder agent
- planner    coder escalates to planner agent
- todos      in-memory todo list (add/update/list)
- notes      scratchpad for discoveries/context


keybindings

<leader>tt  - open tai chat
<leader>tc  - reload config
<leader>ta  - switch agent
<leader>tr  - clear history
<leader>ts  - stop
<C-W><C-T> - toggle chat window


installation

clone or place lua/tai/ into your Neovim config dir
~/.config/nvim/lua/ and add keymaps to init.lua:

  local tai = require("tai")
  tai.setup({})

  vim.keymap.set("n", "<leader>tt", tai.chat)
  vim.keymap.set("n", "<leader>tc", tai.reload)
  vim.keymap.set("n", "<leader>ta", tai.switch_agent)
  vim.keymap.set("n", "<leader>tr", tai.clear_history)
  vim.keymap.set("n", "<leader>ts", tai.stop)
  vim.keymap.set("n", "<C-W><C-T>",
    tai.toggle_chat_window)

or use lazy.nvim, packer, vim-plug, etc.


configuring

tai reads configuration from a .tai JSON file
in your project root:

- provider: provider name (see list above)
- model: model identifier
- options: provider-specific options passed
  to the API (temperature, max_tokens, etc.)
- stream: enable streaming (default false)
- use_tools: enable agent tools (default true)
- think: reasoning effort for supporting models
- rpm: max requests per minute (default 60)
- tpm: max tokens per minute
- provider_tools: provider-side tools array
  (e.g., ["web_search"] for OpenAI)
- system_prompt: override planner system prompt
- custom_prompt: append to system prompt
- allowed_commands: override allowed shell
  commands (default: cat, grep, ag, rg, ls, head,
  tail, wc, diff, sort, uniq, find, file, stat,
  date, echo, tree, pwd, which, type)

example .tai file:

{
  "provider": "groq",
  "model": "llama-3.1-70b-versatile",
  "stream": true,
  "rpm": 30,
  "options": {
    "temperature": 0.7,
    "max_tokens": 4096
  },
  "use_tools": true,
  "allowed_commands": {
    "git": true,
    "npm": true,
    "make": true
  }
}


usage

1. set the correct env var for your provider.
2. create a .tai file at the root of your project.
3. open any file in that project to start tai.
4. use the keybindings to chat, switch agents, etc.

sessions are automatically saved and restored, so
you can close Neovim and pick up where you left off.
use :Tai clear to start fresh.
