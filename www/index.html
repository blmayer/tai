tai.nvim


a minimal Neovim plugin built to bring AI coding
agents directly to your editor.





features

- planner + coder agents for complex tasks
- read/write files and run shell commands
- in-memory todos & notes for tracking progress
- rate limiting (rpm/tpm)
- streaming responses
- extended thinking/reasoning support
- provider-side tools (e.g. web_search)
- safety validation for shell commands
- code folding in the chat buffer


providers

- gemini
- groq
- minimax
- mistral
- ollama / llama.cpp (local)
- openai
- openrouter
- stepfun
- xai
- z.ai
- custom


keybindings

<leader>ti 	- prompt input and send
<leader>tr 	- reset the chat history


installation

clone or place lua/tai/ into your Neovim config dir
~/.config/nvim/lua/ and add keymaps to init.lua:

  vim.g.mapleader = " "

  local tai = require("tai")

  vim.keymap.set("n", "gT", function()
    vim.cmd("set operatorfunc=v:lua.tai_operator_send")
    vim.api.nvim_feedkeys("g@", "n", false)
  end)

  vim.keymap.set("n", "gP", function()
    vim.cmd("set operatorfunc=v:lua.tai_operator_send_with_prompt")
    vim.api.nvim_feedkeys("g@", "n", false)
  end)

  vim.keymap.set("v", "<leader>t", function()
    tai.operator_send("char")
  end, { noremap = true, silent = true })

  vim.keymap.set("i", "<leader>ti", function()
    vim.schedule(tai.prompt_input)
  end, { noremap = true })

  vim.keymap.set("i", "<leader>tf", function()
    vim.schedule(tai.prompt_full_file)
  end, { noremap = true })


configuring

tai reads configuration from a .tai JSON file
in your project root. the following options are
supported:

- model: the model for chat completions
- provider: see providers list above
- options: provider-specific options passed
  to the API (e.g., temperature, max_tokens)
- provider_tools: array of provider-side
  tools (e.g., ["web_search"])
- use_tools: enable/disable agent tools
  (default: true)
- stream: enable streaming responses
  (default: false)
- think: extended thinking/reasoning level
  ("low", "medium", "high")
- rpm: max requests per minute (default: 60)
- tpm: max tokens per minute
- system_prompt: replace default agent prompt
- custom_prompt: append to default prompt
- allowed_commands: override allowed shell
  commands
- auto_approve: if true, auto-run shell commands
  without the y/n/s confirmation prompt (use with care)


agent tools

- read: read files with optional line range
- shell: run allow-listed shell commands
- edit: edit files via old/new replacement
- write: create new files
- send_image: send images for analysis
- coder: (for planner) delegate to coder agent with prompt (gets "coder is working..." ack; coder has own history)
- planner: (for coder) hand back report to planner at end of task (gets ack; planner receives report in its history)
- todos: in-memory todo list (add/update/list)
  with statuses: pending, in_progress, done, cancelled
- notes: in-memory scratchpad (read/write/append)
  for tracking discoveries and context


example .tai file:

{
	"provider": "xai",
	"model": "grok-4.3",
	"options": {
		"temperature": 0.7,
		"max_tokens": 4096
	},
	"stream": true,
	"use_tools": true,
	"rpm": 30,
	"custom_prompt": "Prefer rust for performance.",
	"allowed_commands": {
		"git": true,
		"npm": true,
		"cargo": true
	}
}




usage

make sure the correct env var is set for your provider:
XAI_API_KEY, GROQ_API_KEY, GEMINI_API_KEY, etc.
create a .tai file at the root or your project.
editting any file in that project should start tai.
then just use the keybindings above.
