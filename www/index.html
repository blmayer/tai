tai.nvim + tai-server

a minimal Neovim plugin paired with a unix socket server,
built to bring AI-powered text and code insights directly
to your editor.

the plugin connects to /tmp/tai.sock and sends
selections, prompts, or full files for processing by
a backend server using the groq API.


features

- send motions or visual selections from Neovim
- prompt for input and combine it with text or files
- server caches summaries for faster, smarter replies
- responses show up in a bottom split buffer in Neovim
- requires environment variable GROQ_API_KEY
- simple lua and go code for easy customization


keybindings

gT + motion      - send motion selection
gP + motion      - send motion selection with prompted input
<leader>ti (normal) - prompt input and send
<leader>tf (normal) - prompt input and send with full file


installation

clone or place lua/tai.lua into your Neovim config directory
~/.config/nvim/lua/ and add keymaps to init.lua:

  vim.g.mapleader = " "

  local tai = require("tai")

  vim.keymap.set("n", "gT", function()
    vim.cmd("set operatorfunc=v:lua.tai_operator_send")
    vim.api.nvim_feedkeys("g@", "n", false)
  end)

  vim.keymap.set("n", "gP", function()
    vim.cmd("set operatorfunc=v:lua.tai_operator_send_with_prompt")
    vim.api.nvim_feedkeys("g@", "n", false)
  end)

  vim.keymap.set("v", "<leader>t", function()
    tai.operator_send("char")
  end, { noremap = true, silent = true })

  vim.keymap.set("i", "<leader>ti", function()
    vim.schedule(tai.prompt_input)
  end, { noremap = true })

  vim.keymap.set("i", "<leader>tf", function()
    vim.schedule(tai.prompt_full_file)
  end, { noremap = true })

build and run the go server:

  go build -o tai-server ./path/to/server
  ./tai-server

make sure /tmp/tai.sock exists and
the GROQ_API_KEY environment variable is set.


usage

send text or prompts from Neovim using the
keybindings above. the server will cache file
summaries and respond with enhanced context-aware
answers.
